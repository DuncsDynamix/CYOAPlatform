Everything runs on a single shared engine. Authors use Studio to build experiences once — then the engine delivers them through whichever surface the end-user is on.

The key insight: Stories and Training are not separate systems — they are different visual skins on the same underlying engine. The same node graph can power a consumer story or a corporate learning module.

Shared Infrastructure

TraverseEngine

The AI core that sits beneath everything. It never faces end-users directly — instead it's the engine that drives all the products.

  • Executes the node graph an author has built, one step at a time
  • Calls Claude (Anthropic's AI) to generate unique prose at the moment a reader arrives at each scene
  • Tracks session state — what choices a user has made, what flags have been set, how far through the story they are
  • Routes free-text responses into the correct branch using AI classification
  • Caches generated content so repeated visits are instant
  • Provides a single API that both Stories and Training consume

Authoring Tool · B2B

TraverseStudio

The tool authors use to build experiences. It has a clean, professional UI designed for content designers, learning developers, and narrative authors — not engineers.

  • Build node graphs visually — connect scenes, choices, and checkpoints
  • Configure what the AI should generate at each step via beat instructions
  • Set the world, characters, tone, and learning objectives in the context pack
  • Define choices and branching logic without writing code
  • Publish experiences that deploy instantly to Stories or Training
Authors work in Studio once. The engine runs what they built — repeatedly, for every user, generating fresh content each time.

The Node Graph

Every experience is a directed graph of nodes. Authors connect them in Studio; the engine traverses them at runtime. There are seven node types, each with a distinct purpose.

Fixed
Generated
Choice
Checkpoint
Endpoint
Dialogue

Fixed
Static prose — identical for every user
Generated
AI writes unique prose per user from the beat instruction
Choice
Presents options; open or closed; routes the user to different branches
Checkpoint
Invisible to the user; sets state flags, unlocks branches
Endpoint
Terminal node; AI generates a closing summary of the journey
Dialogue
Multi-turn conversation with an AI actor; detects breakthrough moments
Evaluative
Assesses the user's performance against a rubric using prior context
Authors connect these freely to build any narrative or training structure
Consumer Product · B2C

TraverseStories

The reader-facing product. A book-style CYOA experience where every reader gets a unique, AI-generated story based on the choices they make.

  • Retro book visual identity — feels like reading a novel, not using software
  • Each scene's prose is freshly generated by Claude when the reader arrives
  • Readers make choices (predefined options or free text) that branch the narrative
  • The engine pre-generates upcoming scenes in the background for instant page turns
  • Every playthrough is genuinely different — not just different endings, but different prose throughout
From the reader's perspective: an intimate, beautifully designed book. Behind the scenes: a live AI engine generating their unique story in real time.

Enterprise Product · B2B

TraverseTraining

The L&D render layer. Organisations deploy training scenarios to learners — the same engine, a completely different visual identity and purpose.

  • Professional, corporate visual identity — clean interface suited to workplace settings
  • Learners work through realistic workplace scenarios powered by the same node engine
  • Dialogue nodes enable practice conversations with AI-powered characters
  • Evaluative nodes assess performance against rubrics using the full conversation context
  • Competency profiles track learner progress across a session
  • Organisations can deploy their own branded training modules via Studio
Same engine as Stories. Different visual skin, different use case, different buyer — but the author workflow in Studio is identical.

Summary

The Traverse platform
in four points

One shared engine

TraverseEngine is the AI core that powers everything. Stories and Training are render layers on top — not separate systems. Build once, deploy anywhere.

Authors use Studio

TraverseStudio is the authoring tool. Non-technical authors build node graphs that define the structure — the AI fills in the content at runtime.

Stories for consumers

TraverseStories is the B2C CYOA reader. Book-style interface, AI-generated prose, branching choices — every reader's journey is unique.

Training for organisations

TraverseTraining is the B2B L&D platform. Corporate interface, dialogue practice, competency tracking — same engine, different audience and purpose.

The node graph system — seven node types connected by authors in Studio — is the common language across the entire platform.
